  <action>
Create src/commands/discover.ts following the pattern from src/commands/discuss-milestone.ts.

The command should:

1. **Tool registration**:
   - Name: 'openpaul:discover'
   - Description: 'Research technical options with configurable depth'

2. **Parameters** (Zod schema):
   - phase: number (required) - Phase number
   - topic: string (required) - Research topic/question
   - quick: boolean (optional) - Quick mode (verbal, 2-5 min, no file)
   - deep: boolean (optional) - Deep mode (comprehensive, 1+ hr, requires confirm)
   - confirm: boolean (optional) - Confirmation for deep mode
   - overwrite: boolean (optional) - Overwrite existing DISCOVERY.md

3. **Depth mode logic** (CRITICAL - per CONTEXT.md):

   **Quick mode (--quick flag)**:
   - Return verbal response to console ONLY
   - NO file created
   - Output: Markdown-formatted findings with summary
   - Estimated time: 2-5 minutes

   **Standard mode (default, no flags)**:
   - Create DISCOVERY.md with:
     - 2-3 top options
     - Key tradeoffs
     - Focused scope
   - Estimated time: 15-30 minutes
   - Return: Success message with file location

   **Deep mode (--deep flag)**:
   - Show time estimate: "30-60 minutes"
   - If --confirm not provided:
     - Return confirmation prompt: "Deep research may take 30-60 minutes. Run with --confirm to proceed."
   - If --confirm provided:
     - Create DISCOVERY.md with:
       - 5+ options with edge cases
       - Security/compatibility concerns
       - Detailed examples
     - Estimated time: 1+ hour
   - Return: Success message with file location

4. **Discovery template structure**:
```
# Phase {N}: Discovery - {topic}

**Depth:** {quick|standard|deep}
**Created:** {timestamp}

## Summary

{brief answer to the research question}

## Findings

- {finding 1}
- {finding 2}
...

## Options Considered

### Option 1: {name}
- **Pros:** {benefits}
- **Cons:** {tradeoffs}
- **Verdict:** {recommendation}

### Option 2: {name}
...

## Recommendation

{primary recommendation with rationale}

## References

- {reference 1}
- {reference 2}
```

5. **Error handling**:
   - Both --quick and --deep specified → error with guidance
   - Phase directory not found
   - Deep mode without confirmation → prompt for --confirm
   - Write failure for standard/deep modes
