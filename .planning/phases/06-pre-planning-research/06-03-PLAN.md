Create src/commands/discuss.ts following the pattern from src/commands/discuss-milestone.ts.

The command should:

1. **Tool registration**:
   - Name: 'openpaul:discuss' (also export as 'paulDiscuss' for consistency)
   - Description: 'Explore phase goals and capture planning context'

2. **Parameters** (Zod schema):
   - phase: number (required) - Phase number to discuss
   - domain: string (optional) - Phase boundary description
   - decisions: string (optional) - Comma-separated decision titles
   - specifics: string (optional) - Comma-separated specific ideas
   - deferred: string (optional) - Comma-separated deferred ideas
   - overwrite: boolean (optional) - Overwrite existing CONTEXT.md

3. **Execute function**:
   - Validate phase directory exists (via PrePlanningManager)
   - Parse comma-separated inputs into arrays
   - Generate CONTEXT.md using PrePlanningManager.createContext()
   - Use atomicWrite for file persistence
   - Return formatted output with:
     - Success header (✅ Context Created)
     - Phase number and file location
     - Brief preview of key sections
     - Next steps list

4. **Error handling**:
   - Phase directory not found → formatted error with /openpaul:init suggestion
   - File exists without overwrite → prompt with --overwrite flag
   - Write failure → formatted error with troubleshooting steps

5. **Output format** (per CONTEXT.md decisions):
   - Default: Summary to console + artifact file created
   - Use formatHeader, formatBold, formatList from formatter.ts

Context template structure (from 06-CONTEXT.md):
```
# Phase {N}: {Name} - Context

**Gathered:** {timestamp}
**Status:** Ready for planning

&lt;domain&gt;

{domain content}
&lt;/domain&gt;

&lt;decisions&gt;
## Implementation Decisions
- **{title}:** {description}
&lt;/decisions&gt;

&lt;specifics&gt;
## Specific Ideas
- {idea}
&lt;/specifics&gt;

&lt;deferred&gt;
## Deferred Ideas
- {deferred item} OR "None — discussion stayed within phase scope."
&lt;/deferred&gt;
```
