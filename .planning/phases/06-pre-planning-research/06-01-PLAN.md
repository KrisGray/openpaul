Define these types with matching Zod schemas:

1. **ContextArtifact** - For CONTEXT.md generation:
   - phase: number
   - gathered: string (ISO date)
   - status: string ('Ready for planning' | 'In progress' | 'Complete')
   - domain: string (phase boundary description)
   - decisions: DecisionEntry[] (title, description)
   - specifics: string[] (specific ideas)
   - deferred: string[] (deferred ideas)

2. **AssumptionEntry** - For ASSUMPTIONS.md entries:
   - statement: string
   - validation_status: 'unvalidated' | 'validated' | 'invalidated'
   - confidence: 'high' | 'medium' | 'low'
   - impact: string (what happens if wrong)

3. **AssumptionsArtifact** - For ASSUMPTIONS.md file:
   - phase: number
   - assumptions: AssumptionEntry[]
   - createdAt: string
   - updatedAt: string

4. **IssueEntry** - For ISSUES.md entries:
   - severity: 'critical' | 'high' | 'medium' | 'low'
   - description: string
   - affectedAreas: string[]
   - mitigation: string

5. **IssuesArtifact** - For ISSUES.md file:
   - phase: number
   - issues: IssueEntry[]
   - createdAt: string

6. **DiscoveryArtifact** - For DISCOVERY.md file:
   - phase: number
   - depth: 'quick' | 'standard' | 'deep'
   - summary: string
   - findings: string[]
   - optionsConsidered: string[]
   - recommendation: string
   - references: string[]
   - createdAt: string

Export all types and schemas. Follow existing naming conventions (PascalCase for types, PascalCase + 'Schema' suffix for Zod schemas).
