Create `codecov.yml` in the project root with Codecov configuration:

1. Set coverage precision to 2 decimal places
2. Set coverage range from 50% to 100% with appropriate color coding (red -> yellow -> green)
3. Configure status checks:
   - Project coverage target: auto (uses historical average)
   - Patch coverage: target 80% (new code should have good coverage)
   - Threshold: 1% (allow small decreases without failing)
4. Configure comment behavior:
   - Post comment on PRs
   - Show coverage diff
   - Require head/base commit info
5. Ignore patterns for files that shouldn't affect coverage (e.g., test files, config files)
6. Configure GitHub checks with informational mode to not block PRs initially

Reference Codecov docs for yaml structure. This is a standard config file for a TypeScript/JavaScript npm package.

<task type="auto">
  <name>task 2: Create codecov.yml workflow with workflow_run trigger</name>
  <files>.github/workflows/codecov.yml</files>
  <action>
Create `.github/workflows/codecov.yml` GitHub Actions workflow:

1. **Trigger**: Use `workflow_run` event that triggers when BOTH `CI - Unit Tests` (test.yml) AND `E2E Tests` (e2e-tests.yml) complete successfully:
   - Use `workflow_run` with `types: [completed]`
   - Filter on both workflows by name
   - Only run when `workflow_run.conclusion == 'success'`
   - Use a job condition that checks both test.yml and e2e-tests.yml have succeeded
   
   Note: Since `workflow_run` only triggers on ONE workflow at a time, implement a pattern where:
   - The workflow triggers on any `workflow_run` completion from test.yml or e2e-tests.yml
   - The job checks that BOTH workflows have succeeded before proceeding
   - Use GitHub API to check workflow statuses, OR use a simpler re-usable approach

2. **Alternative approach** (simpler and more reliable): Use `workflow_run` on test.yml completion, but check that e2e-tests.yml also passed:
   - Trigger on test.yml workflow_run completion
   - In the job, use `gh` CLI to verify e2e-tests.yml also succeeded for the same commit SHA
   - Only proceed if both pass

3. **Concurrency**: Use `codecov-${{ github.ref }}` concurrency group to cancel redundant runs

4. **Draft PR filter**: Skip draft PRs using the established pattern

5. **Job steps**:
   - Checkout repository (actions/checkout@v4)
   - Download coverage artifact from test.yml (actions/download-artifact@v4 with `workflow: CI - Unit Tests`)
   - Upload to Codecov (codecov/codecov-action@v4):
     - Use `token: ${{ secrets.CODECOV_TOKEN }}` for authentication
     - Set `file: ./coverage/lcov.info`
     - Enable `fail_ci_if_error: true`
     - Enable verbose output for debugging
     - Set appropriate flags (e.g., `unittests`)

6. **Runner**: ubuntu-latest

Follow existing patterns from test.yml and e2e-tests.yml for action versions and structure.
  </action>

    <manual>Verify workflow triggers on test.yml and e2e-tests.yml completion and uses CODECOV_TOKEN secret</manual>
  </verify>
