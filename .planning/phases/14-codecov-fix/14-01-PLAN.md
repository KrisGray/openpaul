**2. Replace the "Verify both test workflows succeeded" step (lines 40-67) with a wait/retry loop:**

```yaml
      - name: Wait for both test workflows to complete
        if: steps.check_draft.outputs.skip != 'true'
        env:
          GH_TOKEN: ${{ github.token }}
        run: |
          SHA="${{ github.event.workflow_run.head_sha }}"
          MAX_WAIT_MINUTES=30
          POLL_INTERVAL_SECONDS=30
          MAX_ATTEMPTS=$((MAX_WAIT_MINUTES * 60 / POLL_INTERVAL_SECONDS))
          
          echo "Waiting for both test workflows to complete for commit: $SHA"
          echo "Max wait time: $MAX_WAIT_MINUTES minutes"
          
          for attempt in $(seq 1 $MAX_ATTEMPTS); do
            echo "Attempt $attempt of $MAX_ATTEMPTS..."
            
            # Check CI - Unit Tests status
            TEST_STATUS=$(gh run list --workflow "CI - Unit Tests" --commit "$SHA" --limit 1 --json conclusion --jq '.[0].conclusion // "unknown"')
            echo "  CI - Unit Tests: $TEST_STATUS"
            
            # Check E2E Tests status
            E2E_STATUS=$(gh run list --workflow "E2E Tests" --commit "$SHA" --limit 1 --json conclusion --jq '.[0].conclusion // "unknown"')
            echo "  E2E Tests: $E2E_STATUS"
            
            # Handle failure cases
            if [ "$TEST_STATUS" = "failure" ] || [ "$E2E_STATUS" = "failure" ]; then
              echo "One or more workflows failed - skipping coverage upload"
              exit 0
            fi
            
            # Handle cancelled cases
            if [ "$TEST_STATUS" = "cancelled" ] || [ "$E2E_STATUS" = "cancelled" ]; then
              echo "One or more workflows were cancelled - skipping coverage upload"
              exit 0
            fi
            
            # Check if both succeeded
            if [ "$TEST_STATUS" = "success" ] && [ "$E2E_STATUS" = "success" ]; then
              echo "Both test workflows succeeded, proceeding with coverage upload"
              exit 0
            fi
            
            # Still waiting - check if workflows exist at all
            if [ "$TEST_STATUS" = "unknown" ] || [ "$E2E_STATUS" = "unknown" ]; then
              # Check if workflow has started but not completed
              TEST_RUN_ID=$(gh run list --workflow "CI - Unit Tests" --commit "$SHA" --limit 1 --json databaseId --jq '.[0].databaseId // empty')
              E2E_RUN_ID=$(gh run list --workflow "E2E Tests" --commit "$SHA" --limit 1 --json databaseId --jq '.[0].databaseId // empty')
              
              if [ -z "$TEST_RUN_ID" ] || [ -z "$E2E_RUN_ID" ]; then
                echo "One or more workflows have not started yet..."
              else
                echo "Workflows in progress, waiting..."
              fi
            fi
            
            if [ $attempt -lt $MAX_ATTEMPTS ]; then
              echo "Waiting ${POLL_INTERVAL_SECONDS}s before next check..."
              sleep $POLL_INTERVAL_SECONDS
            fi
          done
          
          echo "Timeout reached after $MAX_WAIT_MINUTES minutes"
          echo "One or more workflows did not complete in time"
          exit 0
```

**Key changes:**
- Dual triggers: codecov.yml now triggers when EITHER workflow completes
- Wait loop: Polls every 30 seconds up to 30 minutes max
- Graceful exit: Exits with code 0 (not failure) on timeout - coverage is optional
- Failure detection: Checks for both "failure" and "cancelled" states
- Clear logging: Shows status of each workflow on each poll attempt

**Why exit 0 on timeout:**
- Codecov upload is a quality signal, not a blocking requirement
- Failure to upload coverage should not fail the build
- This matches the behavior of codecov-action with fail_ci_if_error: false
