  <action>
Implement the `/paul:apply` command:

1. **src/commands/apply.ts:**
   - Import `tool` from `@opencode-ai/plugin`
   - Import `z` from `zod`
   - Import `FileManager` from `../storage/file-manager`
   - Import `StateManager` from `../state/state-manager`
   - Import `LoopEnforcer` from `../state/loop-enforcer`
   - Import formatter utilities and progressBar from `../output`

   Args schema:
   - `phase: z.number().int().positive().optional()` — phase number (defaults to current)
   - `plan: z.string().optional()` — plan identifier (defaults to current plan)
   - `task: z.number().int().positive().optional()` — start from specific task
   - `dryRun: z.boolean().optional()` — show tasks without executing

   Execute logic:
   - Create FileManager, StateManager with `context.directory`
   - Get current position
   - If no position: return error "Run /paul:init first"
   - Get phase and plan from args or current state
   - Load plan with `fileManager.readPlan(phase, plan)`
   - If no plan: return error "No plan found. Run /paul:plan first"
   - Create LoopEnforcer
   - If current phase is PLAN: call `enforcer.enforceTransition('PLAN', 'APPLY')`
   - Update state to APPLY: `stateManager.savePhaseState(phase, { phase: 'APPLY', currentPlanId: planId, ... })`

   Output format (from CONTEXT.md):
   - Stage transition announcement: `## 🚀 Starting Apply Phase`
   - Plan summary: name, task count, boundaries
   - For each task:
     - Header: `### Task {n}/{total}: {name}`
     - Files: list of files to modify
     - Action: implementation instructions
     - Verify: how to test
     - Progress bar: `[██░░░] {n}/{total} tasks`
   - Status indicators: ✅ completed, ⏳ in-progress, ⏸️ paused, ❌ failed
   - After all tasks: summary with completion status

   For dryRun mode:
   - Show all tasks with their details
   - Do not execute or update state
   - Add "Dry run - no changes made" footer

   Error handling:
   - Wrap in try/catch
   - On error: save progress, show error with context
   - Return formatted error message with next steps

2. **src/commands/index.ts:**
   - Add `export { paulApply } from './apply'`

3. **src/index.ts:**
   - Import `paulApply` from `./commands/apply`
   - Add `'paul:apply': paulApply` to returned `tool` object

NOTE: This command is informational — it displays plan tasks for the user/assistant to execute. It does NOT actually modify files. The actual task execution is done by the assistant reading the plan output.
