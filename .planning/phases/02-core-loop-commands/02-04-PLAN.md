  <action>
Implement the `/paul:unify` command:

1. **src/commands/unify.ts:**
   - Import `tool` from `@opencode-ai/plugin`
   - Import `z` from `zod`
   - Import `FileManager` from `../storage/file-manager`
   - Import `StateManager` from `../state/state-manager`
   - Import `LoopEnforcer` from `../state/loop-enforcer`
   - Import formatter utilities from `../output`

   Args schema:
   - `phase: z.number().int().positive().optional()` — phase number
   - `plan: z.string().optional()` — plan identifier
   - `notes: z.string().optional()` — additional notes for summary
   - `status: z.enum(['success', 'partial', 'failed']).optional()` — override status
   - `actuals: z.array(z.object({ index: z.number().int().positive(), status: z.enum(['completed','skipped','failed']), notes: z.string().optional() })).optional()`
   - `criteriaResults: z.array(z.object({ criterion: z.string(), status: z.enum(['pass','fail','n/a']), notes: z.string().optional() })).optional()`

   Execute logic:
    - Create FileManager, StateManager with `context.directory`
    - Wrap command execution in try/catch; on error, return `formatGuidedError(...)`
    - Get current position
    - If no position: return guided error "Run /paul:init first"
    - If current phase is not APPLY: return guided error "Must be in APPLY phase. Run /paul:apply first"
    - Load current plan with `fileManager.readPlan()`
    - If no plan: return guided error "No plan found"
    - Create LoopEnforcer, call `enforcer.enforceTransition('APPLY', 'UNIFY')`

   Build Summary (plan vs actual comparison):
    - phaseNumber: from current state
    - planId: from current state
    - completed: count of tasks with actual status completed
    - total: plan.tasks.length
    - status: 'success' (or from args; default to partial/failed if any actuals are skipped/failed)
    - tasks: map plan.tasks to include planned + actual data:
      - planned.verify, planned.done from plan
      - actual.status/notes from args.actuals (default to completed if not provided)
    - criteriaResults: from args or derive `n/a` for each plan criterion
    - createdAt: Date.now()

   Save summary with `fileManager.writeSummary()`
   Only after successful summary write, update state to UNIFY: `stateManager.savePhaseState(phase, { phase: 'UNIFY', ... })`
   Then immediately transition to PLAN for next loop: `stateManager.savePhaseState(phase + 1, { phase: 'PLAN', ... })`
   If any step fails, catch should return formatted guided error and leave existing state unchanged (no partial state writes)

   Output format (from CONTEXT.md):
    - Header: `## 🔗 Loop Closed: ${phase}-${plan}`
    - Summary section with completed/total tasks
    - Status indicator: ✅ success, ⚠️ partial, ❌ failed
    - **Plan vs Actual** section:
      - For each task: planned done/verify vs actual status + notes
    - Criteria results section showing pass/fail/n/a
    - Next steps: "Ready for next loop. Run /paul:plan to start."

2. **src/commands/index.ts:**
   - Add `export { paulUnify } from './unify'`

3. **src/index.ts:**
   - Import `paulUnify` from `./commands/unify`
   - Add `'paul:unify': paulUnify` to returned `tool` object
