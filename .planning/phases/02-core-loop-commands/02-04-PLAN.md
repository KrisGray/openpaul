  <action>
Implement the `/paul:unify` command:

1. **src/commands/unify.ts:**
   - Import `tool` from `@opencode-ai/plugin`
   - Import `z` from `zod`
   - Import `FileManager` from `../storage/file-manager`
   - Import `StateManager` from `../state/state-manager`
   - Import `LoopEnforcer` from `../state/loop-enforcer`
   - Import formatter utilities from `../output`

   Args schema:
   - `phase: z.number().int().positive().optional()` — phase number
   - `plan: z.string().optional()` — plan identifier
   - `notes: z.string().optional()` — additional notes for summary
   - `status: z.enum(['success', 'partial', 'failed']).optional()` — override status

   Execute logic:
   - Create FileManager, StateManager with `context.directory`
   - Get current position
   - If no position: return error "Run /paul:init first"
   - If current phase is not APPLY: return error "Must be in APPLY phase. Run /paul:apply first"
   - Load current plan with `fileManager.readPlan()`
   - If no plan: return error "No plan found"
   - Create LoopEnforcer, call `enforcer.enforceTransition('APPLY', 'UNIFY')`

   Build Summary:
   - phaseNumber: from current state
   - planId: from current state
   - completed: count of tasks (assume all for now)
   - total: plan.tasks.length
   - status: 'success' (or from args)
   - tasks: map plan.tasks to { name, status: 'completed' }
   - createdAt: Date.now()

   Save summary with `fileManager.writeSummary()`
   Update state to UNIFY: `stateManager.savePhaseState(phase, { phase: 'UNIFY', ... })`
   Then immediately transition to PLAN for next loop: `stateManager.savePhaseState(phase + 1, { phase: 'PLAN', ... })`

   Output format (from CONTEXT.md):
   - Header: `## 🔗 Loop Closed: ${phase}-${plan}`
   - Summary section with completed/total tasks
   - Status indicator: ✅ success, ⚠️ partial, ❌ failed
   - Tasks completed list with status
   - Next steps: "Ready for next loop. Run /paul:plan to start."

2. **src/commands/index.ts:**
   - Add `export { paulUnify } from './unify'`

3. **src/index.ts:**
   - Import `paulUnify` from `./commands/unify`
   - Add `'paul:unify': paulUnify` to returned `tool` object
