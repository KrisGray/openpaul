<task type="auto">
  <name>Implement /paul:init command</name>
  <files>
    src/commands/init.ts,
    src/index.ts
  </files>
  <action>
Implement the `/paul:init` command using OpenCode's `tool()` helper:

1. **src/commands/init.ts:**
   - Import `tool` from `@opencode-ai/plugin`
   - Import `z` from `zod`
   - Import `FileManager` from `../storage/file-manager`
   - Import `createDefaultModelConfig` from `../types/model-config`
   - Import formatter utilities from `../output/formatter`

   Args schema:
   - `force: z.boolean().optional()` — reinitialize even if .paul/ exists

   Execute logic:
   - Create FileManager with `context.directory`
   - Check if `.paul/` exists (use `existsSync` from fs)
   - If exists and not force: return warning message with /paul:progress and /paul:init --force suggestions
   - Track created paths in an array (files/dirs created during init)
   - Call `fileManager.ensurePaulDir()` and record the `.paul/` directory if it was newly created
   - Call `fileManager.writeModelConfig(createDefaultModelConfig())` and record the model-config path
   - Create initial state for phase 1 using `fileManager.writePhaseState(1, {...})` and record the state file path
     - phase: 'UNIFY' (ready for new loop)
     - phaseNumber: 1
     - lastUpdated: Date.now()
     - metadata: {}
   - Wrap init in try/catch; on error:
     - Roll back any created files/dirs in reverse order (unlink files, rmdir dirs if empty)
     - Return a `formatGuidedError` message with title "Initialization failed", include error context, suggested fix, and next steps

   Output format (from CONTEXT.md):
   - Header: `## ✅ OpenPAUL Initialized`
   - List created files with descriptions
   - Next steps section suggesting `/paul:plan`

   Use `join` from path for path construction.

2. **src/index.ts:**
   - Import `paulInit` from `./commands/init`
   - Add `'paul:init': paulInit` to returned `tool` object

   DO NOT throw errors — return formatted error messages as strings.
DO NOT use process.cwd() — always use `context.directory`.
  </action>
