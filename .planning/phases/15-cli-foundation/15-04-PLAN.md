Fix the -v flag clash in src/cli.ts with these specific changes:

1. **Line 18 - Remove -v from version flag:**
   Change:
   ```typescript
   .version(pkg.version, '-v, --version', 'output the current version')
   ```
   To:
   ```typescript
   .version(pkg.version, '--version', 'output the current version')
   ```

2. **Line 23 - Add -v short flag to --verbose with counting:**
   Change:
   ```typescript
   .option('--verbose', 'enable verbose output')
   ```
   To:
   ```typescript
   .option('-v, --verbose', 'enable verbose output (use -vv for debug)', countVerbosity, 0)
   ```

3. **Add countVerbosity helper function before the program definition (around line 12):**
   ```typescript
   // Count -v occurrences for verbosity level (0=silent, 1=info, 2+=debug)
   function countVerbosity(value: string, previous: number): number {
     return previous + 1
   }
   ```

4. **Line 32 - Use numeric verbosity value directly:**
   Change:
   ```typescript
   setVerbosity(options.verbose ? 1 : 0)
   ```
   To:
   ```typescript
   setVerbosity(options.verbose)
   ```
   
   Note: Commander's `countVerbosity` function returns a number, so options.verbose will be 0, 1, or 2+ directly.

The setVerbosity() function in src/cli/output.ts already accepts a number (0, 1, 2+) and correctly shows info() at level 1+ and debug() at level 2+.
