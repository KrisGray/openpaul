  <name>Create quality types</name>
  <files>src/types/quality.ts</files>
  <action>
Create src/types/quality.ts with the following types:

1. **UATResult**: 'pass' | 'fail' | 'skip'
2. **UATSeverity**: 'critical' | 'major' | 'minor'
3. **UATCategory**: 'functional' | 'visual' | 'performance' | 'configuration'
4. **UATItem**: Individual test item
   - id: number
   - description: string (the truth from must_haves)
   - result: UATResult
   - notes?: string
   - testedAt?: number (timestamp)

5. **UATIssue**: Failed item with details
   - id: number
   - itemDescription: string
   - status: 'open' | 'fixed' | 'wontfix'
   - severity: UATSeverity
   - category: UATCategory
   - description: string (user-provided issue description)
   - suggestedFix?: string
   - sourcePlanId: string (the plan that created the failing artifact)
   - fixPlanId?: string (assigned by plan-fix)
   - createdAt: number

6. **UAT**: UAT results file content
   - phaseNumber: number
   - planId: string
   - testedAt: number
   - status: 'complete' | 'partial' | 'in-progress'
   - items: UATItem[]
   - summary: { passed: number; failed: number; skipped: number; total: number }

7. **UATIssues**: UAT issues file content
   - phaseNumber: number
   - sourcePlanId: string
   - createdAt: number
   - issues: UATIssue[]

Export Zod schemas for each type. Follow patterns from src/types/plan.ts and src/types/pre-planning.ts.
  </action>

Create src/storage/quality-manager.ts with QualityManager class:

**Methods**:

   - Check for *-SUMMARY.md files in phase directory

   - Find latest *-SUMMARY.md file
   - Parse YAML frontmatter to extract must_haves.truths
   - Return truths array and source plan ID

4. `readUAT(phaseDir: string): UAT | null`
   - Read UAT.md from phase directory
   - Parse and validate with UATSchema

5. `writeUAT(phaseDir: string, uat: UAT): Promise<void>`
   - Write UAT.md with atomic writes
   - Use markdown format from RESEARCH.md

6. `readUATIssues(phaseDir: string): UATIssues | null`
   - Read UAT-ISSUES.md from phase directory
   - Parse and validate with UATIssuesSchema

7. `writeUATIssues(phaseDir: string, issues: UATIssues): Promise<void>`
   - Write UAT-ISSUES.md with atomic writes
   - Use markdown table format from RESEARCH.md

8. `getNextAlphaPlanId(phaseDir: string, basePlanId: string): string`
   - Scan phase directory for existing {phase}-{base}[a-z]*-PLAN.md files
   - Return next alpha suffix (e.g., '01' -> '01a', '01a' -> '01b')

Use PrePlanningManager for phase directory resolution, atomicWrite for file writes, and FileManager patterns for reading.
