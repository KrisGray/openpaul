<task type="auto">
  <name>task 2: Create publish.yml workflow with quality gates and provenance</name>
  <files>.github/workflows/publish.yml</files>
  <action>
Create GitHub Actions workflow file at `.github/workflows/publish.yml` with the following structure:

**Triggers (PUB-01):**
- `release.types: [published]` - triggers on GitHub release published event only

**Quality Gate Job (PUB-02):**
- Job name: `quality-gates`
- Wait for all three workflows to complete successfully for the release commit SHA:
  - `CI - Unit Tests` workflow
  - `E2E Tests` workflow  
  - `Codecov Coverage Upload` workflow
- Use same polling pattern as codecov.yml: gh run list with 30-second intervals, 30-minute max wait
- Fail-fast: exit 1 if any workflow shows `failure` or `cancelled` status
- Only proceed when ALL three show `success`

**Publish Job:**
- Depends on `quality-gates` job
- Runs on `ubuntu-latest`
- Node.js 20.x (match existing workflows)

**Version Sync Step:**
- Extract version from release tag (strip 'v' prefix)
- Validate semver format: must be X.Y.Z with optional prerelease (-beta.1, -next.1)
- If tag doesn't match semver pattern, fail with clear error
- Update package.json version field using jq: `jq --arg v "$VERSION" '.version = $v' package.json > package.json.tmp && mv package.json.tmp package.json`

**Dist-tag Detection Step:**
- Parse release tag for prerelease identifiers
- If tag contains `-beta.` → set DIST_TAG=beta
- If tag contains `-next.` or `-rc.` → set DIST_TAG=next  
- Otherwise → DIST_TAG=latest

**Build Step (PUB-04):**
- Run `npm ci`
- Run `npm run build`

**Publish Step (PUB-03):**
- Configure npm for scoped package: `npm config set //registry.npmjs.org/:_authToken ${NPM_TOKEN}`
- Run: `npm publish --access public --provenance --tag $DIST_TAG`
- Use NPM_TOKEN from repository secrets

**Failure Artifact Upload:**
- On failure, upload `dist/` directory as artifact named `publish-failure-dist`
- Retain for 7 days

**Concurrency:**
- Group: `publish-${{ github.ref }}`
- Cancel in-progress: true

Reference existing workflow patterns from test.yml, e2e-tests.yml, and codecov.yml for consistency in:
- Checkout action with fetch-depth: 1
- Node.js setup with cache: 'npm'
- Draft PR filtering (though releases don't come from PRs, pattern consistency)
- Artifact upload patterns
  </action>
