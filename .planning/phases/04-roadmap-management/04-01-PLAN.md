<task type="auto">
  <name>Task 2: Implement RoadmapManager core class</name>
  <files>src/roadmap/roadmap-manager.ts</files>
  <action>
Create RoadmapManager class with the following methods:

1. Constructor accepting projectRoot: string

2. `resolveRoadmapPath(): string | null` - Find ROADMAP.md in .paul/ or .openpaul/ (follow pattern from resume.ts:285-294)

3. `parsePhases(): PhaseEntry[]` - Parse ROADMAP.md and extract phase entries:
   - Match patterns like "### Phase N:" (for full phase sections)
   - Match table patterns like "| N. | name |" for progress table
   - Extract name, number, and status from both sources

4. `generateDirectoryName(name: string, number: number): string` - Generate directory name:
   - Slugify name: lowercase, replace spaces/special chars with hyphens, remove leading/trailing hyphens
   - Pattern: `{number}-{slugified-name}` (e.g., "05-search-feature")

5. `addPhase(options: AddPhaseOptions): PhaseEntry` - Add new phase:
   - Read ROADMAP.md content
   - Generate directory name
   - Insert phase entry at correct position (after or before reference phase)
   - Renumber subsequent phases in content
   - Create phase directory via fs.mkdirSync
   - Write updated ROADMAP.md
   - Return new PhaseEntry

6. `canRemovePhase(phaseNumber: number, statePath: string): RoadmapValidationResult` - Validate removal:
   - Check ROADMAP.md exists
   - Check phase exists
   - Read STATE.md to get current phase (block if removing current phase)
   - Check phase status is not 'complete' or 'in-progress'
   - Return validation result with any errors

7. `removePhase(phaseNumber: number): RemovePhaseResult` - Remove phase:
   - Read ROADMAP.md content
   - Remove phase entry (section and table row)
   - Renumber all subsequent phases (decrement by 1)
   - Delete phase directory via fs.rmSync with recursive:true
   - Write updated ROADMAP.md
   - Return result with removed phase and renumbered phases

Use fs module (readFileSync, writeFileSync, existsSync, mkdirSync, rmSync, readdirSync).
Use join from path for file paths.
  </action>
