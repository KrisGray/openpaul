Create src/commands/status.ts with paulStatus tool:

Command definition:
- Description: "View current project status and loop position"
- Args: verbose (optional boolean) for detailed output

Execute logic:
1. Create StateManager instance with context.directory
2. Create FileManager instance with context.directory
3. Create SessionManager instance with context.directory

4. Check initialization:
   - If .paul/ doesn't exist, return "Not initialized" with instructions to run /openpaul:init

5. Get current position: stateManager.getCurrentPosition()
6. Return error if no active state (not initialized)

7. Load current session: sessionManager.loadCurrentSession()

8. Format loop visualization:
   - Reuse formatLoopVisual function from progress.ts (import and reuse)
   - Or implement inline if easier:
     - Loop phases: ['PLAN', 'APPLY', 'UNIFY']
     - For each phase:
       - If current phase: `◉ ${phase}` (filled circle)
       - If before current: `✓ ${phase}` (checkmark)
       - If after current: `○ ${phase}` (open circle)
     - Join with ' → ' separator

9. Format status output:
   - Header: "📍 OpenPAUL Status"
   - Loop visualization on its own line
   - Phase: position.phaseNumber
   - Current Stage: position.phase
   - If verbose, show: Last updated timestamp from phase state

10. Show session info (if paused):
    - If session exists:
      - Calculate hours since pause: (Date.now() - session.pausedAt) / (1000 * 60 * 60)
      - Show staleness warning if > 24 hours: "⚠️ {Math.round(hoursAgo)}h old"
      - Show Session ID: session.sessionId
      - Show Paused: new Date(session.pausedAt).toLocaleString()
    - If no session: "No active session"

11. Show plan progress (if in APPLY phase):
    - Load phase state: stateManager.loadPhaseState(position.phaseNumber)
    - Get current plan: fileManager.readPlan(position.phaseNumber, currentPlanId)
    - If plan exists:
      - Count total tasks: plan.tasks.length
      - Count completed tasks: from phase state metadata (task completion tracking)
      - Show progress: progressBar(completed, total)
      - Show "X/Y plans complete" for phase
    - If no plan: "No active plan"

12. Show next action:
    - Get next action: stateManager.getRequiredNextAction(position.phase)
    - Display next action

13. Verbose output (if verbose flag set):
    - Show additional details:
      - Current timestamp
      - File paths: .openpaul/STATE.md, .openpaul/ROADMAP.md
      - Quick command list: /openpaul:plan, /openpaul:apply, /openpaul:unify, /openpaul:pause, /openpaul:resume

14. Wrap in try/catch, return formatted error on failure (don't throw)

Follow existing command pattern from src/commands/progress.ts. Reuse formatLoopVisual if possible, or implement similar logic.

Ensure loop visualization matches existing format from progress.ts:
- Current phase: ◉
- Completed phases: ✓
- Future phases: ○
- Separator: →
