Create src/output/diff-formatter.ts with diff formatting functions:

Install diff package first (in task action):
- Run: npm install diff

Functions:

1. formatFileDiff(oldContent: string, newContent: string): string
   - Import diffLines from 'diff'
   - Generate diff: diffLines(oldContent, newContent)
   - Map each part to formatted line:
     - If part.added: prefix with '+ ' (green)
     - If part.removed: prefix with '- ' (red)
     - Otherwise: prefix with '  ' (neutral)
   - Join lines with newline
   - Return formatted diff string

2. formatDiff(changes: Array<{ type: 'modified' | 'added' | 'deleted', filePath: string, diff?: string }>): string
   - Build summary header: "Changes since pause"
   - Count changes by type: modified, added, deleted
   - Show summary: "X files changed: Y modified, Z added, W deleted"
   - For each change:
     - Show file path with type indicator (M/A/D)
     - If diff provided and < 50 lines, show formatted diff
     - Otherwise, show summary: "(large file, run verbose for full diff)"
   - Return formatted string with sections separated by newlines

3. formatStalenessWarning(hoursAgo: number): string
   - Return warning if hours > 24: "⚠️ Session paused {Math.round(hoursAgo)} hours ago"
   - Otherwise return empty string

Note: This is a simplified diff formatter. For now, we'll track changed files but not generate actual file diffs (since we don't track file checksums in session state yet). In a future enhancement, we can add file content tracking to SessionState and generate actual diffs.

For now, formatDiff will show a summary but no actual file content diffs.

Create src/commands/resume.ts with paulResume tool:

Command definition:
- Description: "Resume paused development session"
- No args required

Execute logic:
1. Create SessionManager instance with context.directory
2. Create StateManager instance with context.directory

3. Load current session: sessionManager.loadCurrentSession()
4. Return error if no session found (not paused)
5. Validate session: sessionManager.validateSessionState(session.sessionId)
6. Return error if validation fails

7. Check staleness:
   - Calculate hours since pause: (Date.now() - session.pausedAt) / (1000 * 60 * 60)
   - Show staleness warning if > 24 hours

8. Format session summary:
   - Header: "📋 Session Resume"
   - Session ID
   - Paused timestamp
   - Current phase: session.phaseNumber - session.phase
   - Work in progress: formatList(session.workInProgress)
   - Next steps: formatList(session.nextSteps)

9. Check for changes (simplified version):
   - For now, we don't track file checksums in session state
   - Show message: "File changes not tracked in current session state"
   - In future enhancement, add file checksums to SessionState and generate actual diffs

10. Format loop visualization:
    - Use formatLoopVisual from progress command (reuse or reimplement)
    - Show current loop position

11. Show next action:
    - Get next action: stateManager.getRequiredNextAction(session.phase)
    - Display next action

12. Return formatted output with:
    - Session summary
    - Staleness warning (if applicable)
    - Loop visualization
    - Next action
    - Instructions: "Run /openpaul:status for full project status"

13. Wrap in try/catch, return formatted error on failure (don't throw)

Note: For this plan, we're implementing a simplified resume that doesn't actually generate file diffs (since we don't track file checksums yet). The focus is on loading session state and showing basic session info. Future enhancement will add file checksum tracking and actual diff generation.
