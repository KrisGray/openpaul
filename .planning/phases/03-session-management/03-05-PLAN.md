<task type="auto">
  <name>Implement /openpaul:handoff command</name>
  <files>src/commands/handoff.ts</files>
  <action>
Create src/commands/handoff.ts with paulHandoff tool:

Command definition:
- Description: "Create explicit handoff document for team collaboration"
- Args: none

Execute logic:
1. Create StateManager instance with context.directory
2. Create FileManager instance with context.directory
3. Create SessionManager instance with context.directory

4. Get current position: stateManager.getCurrentPosition()
5. Return error if no active state (not initialized)

6. Load current session: sessionManager.loadCurrentSession()
7. If no session exists, create temporary session state on-the-fly:
   - sessionId: sessionManager.generateSessionId()
   - createdAt: Date.now()
   - pausedAt: Date.now()
   - phase: position.phase
   - phaseNumber: position.phaseNumber
   - currentPlanId: position.currentPlanId (from state metadata)
   - workInProgress: [] (empty)
   - nextSteps: [stateManager.getRequiredNextAction(position.phase)]
   - metadata: {}

8. Read HANDOFF.md template:
   - Template path: join(__dirname, '../../templates/HANDOFF.md')
   - Read template content: readFileSync(templatePath, 'utf-8')

9. Create replacement object with all template variables:
   - timestamp: new Date().toISOString()
   - session_id: sessionState.sessionId
   - status: 'paused' if paused, 'active' if on-the-fly
   - project_name: extract from .openpaul/model-config.json or 'OpenPAUL Project'
   - core_value: 'Enforce the PLAN → APPLY → UNIFY loop with mandatory reconciliation'
   - version: extract from package.json or '1.0.0'
   - phase_number: String(sessionState.phaseNumber)
   - total_phases: extract from ROADMAP.md (count phases) or 'N'
   - phase_name: extract phase name from ROADMAP.md for phaseNumber or 'Unknown'
   - plan_id: sessionState.currentPlanId || 'none'
   - plan_status: sessionState.phase
   - accomplished_list: formatList([]) (empty for now, can be enhanced later)
   - in_progress_list: formatList(sessionState.workInProgress)
   - next_action: sessionState.nextSteps[0] || 'No next step'
   - following_action: 'Continue with next plan'
   - current_plan_path: sessionState.currentPlanId
     ? `.paul/phases/${sessionState.phaseNumber}-${sessionState.currentPlanId}-PLAN.json`
     : 'none'
   - plan_purpose: sessionState.currentPlanId ? 'Current active plan' : 'No active plan'
   - plan_mark: sessionState.phase === 'PLAN' ? '●' : '○'
   - apply_mark: sessionState.phase === 'APPLY' ? '●' : sessionState.phase === 'PLAN' ? '○' : '○'
   - unify_mark: sessionState.phase === 'UNIFY' ? '●' : sessionState.phase === 'APPLY' ? '○' : '○'

10. Replace all template variables:
    - For each key-value pair in replacement object:
      - Replace all occurrences of `{{key}}` in template with value
      - Use global regex: template.replace(new RegExp(`{{${key}}}`, 'g'), value)

11. Write HANDOFF.md:
    - Output path: join(context.directory, '.openpaul', 'HANDOFF.md')
    - Use atomicWrite from src/storage/atomic-writes.ts to write file

12. Format success output:
    - Header: "✅ Handoff Created"
    - HANDOFF.md location: .openpaul/HANDOFF.md
    - Session ID: sessionState.sessionId
    - Instructions:
      - "Review and edit HANDOFF.md before sharing"
      - "Share HANDOFF.md with team members"
      - "To resume: /openpaul:resume"

13. Wrap in try/catch, return formatted error on failure (don't throw)

Follow existing command pattern from src/commands/pause.ts. Reuse template replacement logic from pause command.

Note: This command can create handoff even if no session is paused. It creates a temporary session state on-the-fly based on current project state.
