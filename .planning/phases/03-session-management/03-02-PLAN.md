<task type="auto">
  <name>Implement /openpaul:pause command</name>
  <files>src/commands/pause.ts</files>
  <action>
Create src/commands/pause.ts with paulPause tool:

Command definition:
- Description: "Pause current development session and save context"
- No args required

Execute logic:
1. Create StateManager instance with context.directory
2. Create FileManager instance with context.directory
3. Create SessionManager instance with context.directory
4. Get current position: stateManager.getCurrentPosition()
5. Return error if no active state (not initialized)

6. Check for existing session: sessionManager.getCurrentSessionId()
7. If exists, load session with sessionManager.loadCurrentSession()
8. Calculate hours since pause: (Date.now() - pausedAt) / (1000 * 60 * 60)
9. If recent (< 24 hours), return warning with confirmation prompt

10. Capture session state:
    - sessionId: sessionManager.generateSessionId()
    - createdAt: Date.now()
    - pausedAt: Date.now()
    - phase: position.phase
    - phaseNumber: position.phaseNumber
    - currentPlanId: position.currentPlanId (from state metadata)
    - workInProgress: [] (empty for now, will be extracted in later enhancement)
    - nextSteps: [stateManager.getRequiredNextAction(position.phase)]
    - metadata: {} (reserved for future use)

11. Save session: await sessionManager.saveSession(sessionState)

12. Generate HANDOFF.md:
    - Read template from src/templates/HANDOFF.md (readFileSync)
    - Create replacement object with:
      - timestamp: new Date().toISOString()
      - session_id: sessionState.sessionId
      - status: 'paused'
      - project_name: extract from .openpaul/model-config.json or 'Unknown'
      - core_value: 'Enforce the PLAN → APPLY → UNIFY loop with mandatory reconciliation'
      - version: '1.0.0' (or from package.json)
      - phase_number: String(sessionState.phaseNumber)
      - phase_name: extract from ROADMAP.md or 'Unknown'
      - plan_id: sessionState.currentPlanId || 'none'
      - plan_status: sessionState.phase
      - accomplished_list: formatList([]) (empty for now)
      - in_progress_list: formatList(sessionState.workInProgress)
      - next_action: sessionState.nextSteps[0] || 'No next step'
      - following_action: 'Continue with next plan'
      - current_plan_path: sessionState.currentPlanId
        ? `.paul/phases/${sessionState.phaseNumber}-${sessionState.currentPlanId}-PLAN.json`
        : 'none'
      - plan_purpose: 'Current active plan'
      - plan_mark: sessionState.phase === 'PLAN' ? '●' : '○'
      - apply_mark: sessionState.phase === 'APPLY' ? '●' : sessionState.phase === 'PLAN' ? '○' : '○'
      - unify_mark: sessionState.phase === 'UNIFY' ? '●' : sessionState.phase === 'APPLY' ? '○' : '○'
    - Replace all {{variable}} placeholders in template
    - Write HANDOFF.md to .openpaul/HANDOFF.md (use fileManager.writeFile or write with atomicWrite)

13. Format success output:
    - Header: "✅ Session Paused"
    - Session ID
    - Paused timestamp
    - Current phase and loop position
    - HANDOFF.md location
    - Next steps: /openpaul:resume to continue, /openpaul:status to check position

14. Wrap in try/catch, return formatted error on failure (don't throw)

Follow existing command pattern from src/commands/init.ts.
  </action>

npm run build (TypeScript compilation succeeds)
  </verify>
