  <name>Implement /paul:plan command</name>
  <files>

  <action>
Implement the `/paul:plan` command:

1. **src/commands/plan.ts:**
   - Import `tool` from `@opencode-ai/plugin`
   - Import `z` from `zod`
   - Import `FileManager` from `../storage/file-manager`
   - Import `StateManager` from `../state/state-manager`
   - Import `LoopEnforcer` from `../state/loop-enforcer`
   - Import formatter utilities from `../output/formatter`

   Args schema:
   - `phase: z.number().int().positive()` — phase number
   - `plan: z.string()` — plan identifier (e.g., "01", "02")
   - `tasks: z.array(z.object({ name, files, action, verify, done }))` — tasks to add
   - `criteria: z.array(z.string()).optional()` — plan success criteria
   - `boundaries: z.array(z.string()).optional()` — scope boundaries / exclusions
   - `verbose: z.boolean().optional()` — show full task details
   - `task: z.number().int().positive().optional()` — show a single task view

   Execute logic:
   - Create FileManager and StateManager with `context.directory`
   - Get current position with `stateManager.getCurrentPosition()`
   - If no position: return error "Run /paul:init first"
   - Create LoopEnforcer, call `enforceCanStartNewLoop(currentPhase)`
   - Check if plan already exists with `fileManager.planExists()`
   - If exists: return error "Plan already exists"
   - Build Plan object:
      - phase: `${phase}`
      - plan: `${plan}` (zero-padded)
      - type: 'execute' (default)
      - wave: 1 (auto-detected from depends_on, default 1)
      - depends_on: [] (empty for now)
      - files_modified: flatten all task.files
      - autonomous: true (no checkpoints by default)
      - requirements: [] (empty for now)
      - tasks: from args
      - criteria: from args or []
      - boundaries: from args or []
      - taskDependencies: auto-detected from file overlaps (see below)
      - executionGraph: derived wave graph from taskDependencies (see below)
      - must_haves: { truths: [], artifacts: [], key_links: [] } (empty for now)
   - Auto-detect dependencies:
     - Build a map of task index -> file list
     - If task B shares any file with task A and A index < B index, then B depends on A
     - Store as `taskDependencies` (1-based indexes)
   - Build `executionGraph` from dependencies (topological grouping into waves)
     - Output format: `[[1,3],[2]]` meaning tasks 1 and 3 can run in parallel, then task 2
    - Call `fileManager.ensurePhasesDir()`
    - Call `fileManager.writePlan(phase, plan, planObject)`
    - Update state: `stateManager.savePhaseState(phase, { phase: 'PLAN', ... })`

   Output format (from CONTEXT.md):
    - Header: `# 📋 Plan: ${phase}-${plan}`
    - Meta: `**Type:** ${type} | **Wave:** ${wave} | **Tasks:** ${tasks.length}`
    - Default view: numbered task list with name + one-line description
    - Default view also includes:
      - Boundaries (if present)
      - Criteria (if present)
      - Execution graph line: `Execution: [1,3] → [2]`
    - Verbose view: full task details with files, verify, done, criteria/boundaries, dependencies
    - Task view (--task N): full details for a single task + its dependencies

   Use try/catch to catch errors and return formatted error messages.
   Use `formatGuidedError` from `src/output/error-formatter.ts` to include:
   - message, context, suggested fix, next steps
   - command references (/paul:help, /paul:progress)

   Retry/rollback behavior (from CONTEXT.md):
   - Validation errors (zod): return guided error with suggested fix; no retry
   - State errors (LoopEnforcer / missing init): return guided error; no retry
   - File system errors: retry once with small backoff; on failure, rollback (delete any partially written plan file)
   - Unknown errors: return guided error and keep state unchanged

   DO NOT throw errors — return formatted strings.

2. **src/commands/index.ts:**
   - Add `export { paulPlan } from './plan'`

3. **src/index.ts:**
   - Import `paulPlan` from `./commands/plan`
   - Add `'paul:plan': paulPlan` to returned `tool` object
