  <name>task 1: Implement /openpaul:discuss-milestone command</name>
  <files>src/commands/discuss-milestone.ts</files>
  <action>
Create discuss-milestone command following template patterns from pause.ts and handoff.ts:

1. Define Zod args schema:
   - name: string (milestone name to plan, e.g., "v2.0")
   - scope: optional string (milestone scope/description)
   - features: optional string (comma-separated feature list)
   - phases: optional string (comma-separated phase numbers)
   - constraints: optional string (constraints or limitations)
   - overwrite: optional boolean (overwrite existing MILESTONE-CONTEXT.md)

2. Command behavior (per CONTEXT.md decisions):
   - Purpose: Plan UPCOMING milestone (not documenting current)
   - Location: .planning/MILESTONE-CONTEXT.md (single file)
   - Interactive prompts if args not provided
   
3. Implementation flow:
   a. Check if MILESTONE-CONTEXT.md exists
   b. If exists and no --overwrite, prompt user
   c. Collect context via prompts or CLI args:
      - Milestone name
      - Scope/goals
      - Features to include
      - Phase mapping (which phases for which features)
      - Constraints/timeline/deadlines
   d. Generate MILESTONE-CONTEXT.md from template
   e. Format success output with file location

4. MILESTONE-CONTEXT.md template structure:
   ```markdown
   # Milestone Context: {name}
   
   **Gathered:** {date}
   **Status:** Planning
   
   ## Goals
   
   {scope}
   
   ## Features
   
   {features list}
   
   ## Phase Mapping
   
   {phase to feature mapping}
   
   ## Constraints
   
   {constraints}
   
   ## Open Questions
   
   {unresolved items}
   
   ---
   
   *Context for upcoming milestone*
   *Created: {date}*
   ```

5. Error handling:
   - File exists → prompt to overwrite or cancel
   - Invalid milestone name → validation error
   - Write failure → return formatted error

Register command in src/index.ts as `paulDiscussMilestone`.
