  <action>
Create src/commands/verify.ts implementing the /openpaul:verify command.

**Parameters** (Zod schema):
- phase: z.number().int().positive() - Phase number (required)
- item: z.number().int().positive().optional() - Item number to test
- result: z.enum(['pass', 'fail', 'skip']).optional() - Test result
- notes: z.string().optional() - Notes for pass/fail
- severity: z.enum(['critical', 'major', 'minor']).optional() - For failures
- category: z.enum(['functional', 'visual', 'performance', 'configuration']).optional() - For failures

**Behavior** (from CONTEXT.md locked decisions):

1. **No SUMMARY.md**: Show error:
   "SUMMARY.md not found. Run /gsd-execute-phase first to generate it."
   Include next steps list.

2. **No --item flag**: Display checklist:
   - Show numbered list of truths from must_haves
   - Show progress (X/Y tested, Z passed)
   - Show instruction: "Test an item: /openpaul:verify --phase N --item M --result pass|fail|skip"

3. **With --item and --result**:
   - Update UAT.md with result
   - If pass/skip: show updated progress
   - If fail: validate --severity and --category are provided
     - Prompt for --notes with issue description
     - Show prompt: "Provide issue description with --notes flag"

4. **Failed item flow**:
   - Create/update UAT-ISSUES.md with issue details
   - Auto-link to source plan ID
   - Show: "Issue recorded. Run /openpaul:plan-fix to create fix plan."

5. **Completion**: When all items tested:
   - Show summary: X passed, Y failed, Z skipped
   - Note UAT.md and UAT-ISSUES.md locations
   - If failures exist: "Run /openpaul:plan-fix to address issues"

**Output files** (markdown format from RESEARCH.md):
- UAT.md: Results table with pass/fail/skip
- UAT-ISSUES.md: Table + details for failed items

Follow confirmation flow patterns from complete-milestone.ts. Use formatter utilities for output.
