  <action>
Create src/commands/research.ts following the pattern from src/commands/discuss-milestone.ts.

The command should:

1. **Tool registration**:
   - Name: 'openpaul:research'
   - Description: 'Execute research on user-specified topics with verification'

2. **Parameters** (Zod schema):
   - phase: number (required) - Phase number for context
   - query: string (required) - Research question or topic
   - depth: enum (optional) - 'quick' | 'standard' | 'deep' (default: 'standard')
   - verify: boolean (optional) - Include verification of findings (default: true)
   - sources: string (optional) - Comma-separated preferred sources
   - overwrite: boolean (optional) - Overwrite existing RESEARCH.md

3. **Execute function**:
   - Validate phase directory exists
   - Execute research based on depth mode
   - Collect findings with sources
   - Assign confidence levels (HIGH/MEDIUM/LOW) per finding
   - Verify findings if --verify flag set
   - Generate RESEARCH.md using ResearchManager
   - Return formatted output with:
     - Success header (✅ Research Complete)
     - Query summary
     - Confidence distribution
     - File location
     - Key findings preview

4. **Research template structure**:
```
# Phase {N}: Research - {query}

**Depth:** {depth}
**Verified:** {yes/no}
**Created:** {timestamp}
**Confidence:** {overall confidence}

## Summary

{brief answer to research query}

## Findings

### {Theme 1}

- **Finding:** {description}
- **Confidence:** HIGH|MEDIUM|LOW
- **Source:** {source}
- **Verification:** {verification notes}

### {Theme 2}
...

## Confidence Breakdown

| Level | Count | Topics |
|-------|-------|--------|
| HIGH | {n} | {topics} |
| MEDIUM | {n} | {topics} |
| LOW | {n} | {topics} |

## Sources

- {source 1}
- {source 2}
...

## Recommendations

{actionable recommendations based on findings}
```

5. **Error handling**:
   - Phase directory not found
   - Empty query provided
   - Research execution failure (partial results saved)
   - Write failure
