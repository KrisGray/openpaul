<task type="auto">
  <name>task 1: Create /openpaul:consider-issues command</name>
  <files>src/commands/consider-issues.ts</files>
  <action>
Create src/commands/consider-issues.ts following the pattern from src/commands/discuss-milestone.ts.

The command should:

1. **Tool registration**:
   - Name: 'openpaul:consider-issues'
   - Description: 'Identify potential blockers and risks'

2. **Parameters** (Zod schema):
   - phase: number (required) - Phase number
   - issues: string (required) - Comma-separated issue descriptions
   - severities: string (optional) - Comma-separated severity levels (default: 'medium')
   - areas: string (optional) - Comma-separated affected areas (pipe-separated for multiple per issue)
   - mitigations: string (optional) - Comma-separated mitigation strategies
   - append: boolean (optional) - Append to existing file
   - overwrite: boolean (optional) - Overwrite existing file

3. **Execute function**:
   - Parse comma-separated inputs into IssueEntry[] array
   - Validate severity values (critical, high, medium, low)
   - Parse pipe-separated areas for each issue
   - Sort issues by severity (critical → low)
   - Generate ISSUES.md using PrePlanningManager.createIssues()
   - Return formatted output with:
     - Success header (✅ Issues Identified)
     - Count by severity level
     - File location
     - Summary of critical/high issues

4. **Error handling**:
   - Invalid severity value → error with valid options
   - Phase directory not found
   - File write failure

5. **Issues template structure** (from 06-CONTEXT.md):
```
# Phase {N}: Issues and Risks

**Created:** {timestamp}
**Status:** Open

## Critical Issues

### 1. {description}
- **Severity:** Critical
- **Affected Areas:** {area1}, {area2}
- **Mitigation:** {suggested mitigation}

## High Priority Issues

### 2. {description}
...

## Medium Priority Issues

...

## Low Priority Issues

...

## Summary

| Severity | Count |
|----------|-------|
| Critical | {n} |
| High | {n} |
| Medium | {n} |
| Low | {n} |
```
