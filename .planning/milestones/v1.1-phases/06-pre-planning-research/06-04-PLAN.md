Create src/commands/assumptions.ts following the pattern from src/commands/discuss-milestone.ts.

The command should:

1. **Tool registration**:
   - Name: 'openpaul:assumptions'
   - Description: 'Capture and validate project assumptions'

2. **Parameters** (Zod schema):
   - phase: number (required) - Phase number
   - assumptions: string (required) - Comma-separated assumption statements
   - statuses: string (optional) - Comma-separated validation statuses (default: 'unvalidated')
   - confidences: string (optional) - Comma-separated confidence levels (default: 'medium')
   - impacts: string (optional) - Comma-separated impact descriptions
   - append: boolean (optional) - Append to existing file
   - overwrite: boolean (optional) - Overwrite existing file

3. **Execute function**:
   - Parse comma-separated inputs into AssumptionEntry[] array
   - Map each assumption to its status/confidence/impact (parallel arrays)
   - Generate ASSUMPTIONS.md using PrePlanningManager.createAssumptions()
   - Handle append vs overwrite logic
   - Return formatted output with:
     - Success header (✅ Assumptions Captured)
     - Count of assumptions captured
     - File location
     - Brief preview of first few assumptions

4. **Error handling**:
   - Phase directory not found
   - Invalid validation status (not unvalidated/validated/invalidated)
   - Invalid confidence level (not high/medium/low)
   - Mismatched array lengths for parallel inputs

Assumptions template structure (from 06-CONTEXT.md):
```
# Phase {N}: Assumptions

**Created:** {timestamp}
**Updated:** {timestamp}

## Assumptions

| Statement | Status | Confidence | Impact |
|-----------|--------|------------|--------|
| {statement} | {validation_status} | {confidence} | {impact} |

## Validation Notes
```
