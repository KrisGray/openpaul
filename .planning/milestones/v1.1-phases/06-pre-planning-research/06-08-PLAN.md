  <action>
Create src/commands/research-phase.ts following the pattern from src/commands/discuss-milestone.ts.

The command should:

1. **Tool registration**:
   - Name: 'openpaul:research-phase'
   - Description: 'Auto-detect phase unknowns and research them in parallel'

2. **Parameters** (Zod schema):
   - phase: number (required) - Phase number to research
   - topics: string (optional) - Comma-separated specific topics (auto-detected if not provided)
   - maxAgents: number (optional) - Maximum parallel agents (default: 4, max: 4)
   - depth: enum (optional) - 'quick' | 'standard' | 'deep' (default: 'standard')
   - continue: boolean (optional) - Continue even if some agents fail (default: true)
   - overwrite: boolean (optional) - Overwrite existing RESEARCH.md

3. **Execute function flow**:
   
   a) **Phase analysis** (if topics not provided):
      - Load phase CONTEXT.md or ROADMAP.md phase description
      - Extract technical terms, libraries, patterns mentioned
      - Identify "unknowns" (terms without definitions, unfamiliar patterns)
      - Generate up to 4 research topics
   
   b) **Agent spawning simulation**:
      - Create AgentStatus for each topic
      - Display agent status dashboard:
        ```
        ## Research Progress
        
        🔄 **Topic 1: Library X patterns** — running
           Investigating core API patterns...
        ⏳ **Topic 2: Authentication flow** — spawning
        ✅ **Topic 3: Database schema** — complete
           Found 3 schema patterns, HIGH confidence
        ❌ **Topic 4: External API** — failed
           Error: API documentation not accessible
        
        **Progress:** 2/4 agents complete
        ```
   
   c) **Result aggregation**:
      - Collect findings from all agents
      - Organize by theme (not by agent)
      - Note failed agents in report
      - Create single synthesized RESEARCH.md
   
   d) **Output**:
      - Success/failure summary
      - Agent completion status
      - File location
      - Key themes discovered

4. **Research-phase template structure**:
```
# Phase {N}: Phase Research

**Topics Researched:** {n}
**Agents Spawned:** {n}
**Completed:** {n} successful, {n} failed
**Created:** {timestamp}

## Research Summary

{synthesized overview of all findings}

## Agent Results

| Topic | Status | Confidence | Key Finding |
|-------|--------|------------|-------------|
| {topic} | ✅ Complete | HIGH | {summary} |
| {topic} | ❌ Failed | - | Error: {error} |

## Findings by Theme

### Theme: {theme name}

**From:** {topic 1}, {topic 2}

- {finding 1}
- {finding 2}
...

### Theme: {another theme}
...

## Failed Agents

{topic}: {error message}
{recommendation for manual research}

## Recommendations

{consolidated recommendations for phase planning}
```

5. **Per CONTEXT.md requirements**:
   - Max 4 agents
   - Continue with partial results
   - Agent status dashboard
   - Organize by theme
   - Failed agents noted in report

6. **Error handling**:
   - Phase not found in ROADMAP.md
   - No topics detected and none provided
   - All agents fail (still save partial results)
   - Write failure
