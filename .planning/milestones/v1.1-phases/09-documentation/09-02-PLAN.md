  <action>
For each storage manager that resolves paths, implement the dual-path resolution pattern:

1. In file-manager.ts:
   - Update the constructor to track both paths:
     ```typescript
     this.openPaulDir = join(projectRoot, '.openpaul')
     this.paulDir = join(projectRoot, '.paul')
     ```
   - Add/update resolvePath methods to check .openpaul/ first, then .paul/:
     ```typescript
     private resolvePath(filename: string): string {
       const openPaulPath = join(this.openPaulDir, filename)
       if (existsSync(openPaulPath)) return openPaulPath
       return join(this.paulDir, filename)
     }
     ```
   - When WRITING files, always write to .openpaul/
   - When READING files, check .openpaul/ first, fall back to .paul/

2. Apply the same pattern to:
   - config-manager.ts (config path resolution)
   - flows-manager.ts (flows path resolution)
   - session-manager.ts (session state path resolution)
   - pre-planning-manager.ts (phase directory resolution)
   - research-manager.ts (research files path resolution)

3. Update JSDoc comments to document the dual-path behavior.

Note: milestone-manager.ts and roadmap/roadmap-manager.ts already have this pattern - verify they're consistent.
  </action>
