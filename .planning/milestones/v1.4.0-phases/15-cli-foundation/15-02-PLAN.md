Add an action handler to the commander program that captures and validates the options.

From CONTEXT.md locked decisions:
- Bare `npx openpaul` starts interactive mode
- Providing flags (`--name`, `--path`, `--preset`) skips prompts for those values
- `--interactive` flag explicitly enables interactive prompts

Add an action handler that:
1. Receives the parsed options object
2. Logs the options (for now, actual scaffolding is Phase 16)

```typescript
.action(async (options) => {
  // Options available: options.path, options.name, options.interactive, options.verbose
  // For now, just log that we received the command
  // Phase 16 will implement actual scaffolding
  console.log(`OpenPAUL CLI initialized`)
  console.log(`Path: ${options.path}`)
  if (options.name) {
    console.log(`Name: ${options.name}`)
  }
})
```

Also add the --interactive and --verbose options for future use:
```typescript
.option('-i, --interactive', 'force interactive mode')
.option('--verbose', 'enable verbose output')
```
