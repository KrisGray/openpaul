OpenPAUL

Plan-Apply-Unify Loop — Structured AI-assisted development for OpenCode

Get Started View on GitHub

Why OpenPAUL?

🔄 Loop Integrity

Every plan closes with UNIFY. No orphan plans. Every unit of work gets reconciled.

🎯 In-Session Context

Execution stays in-session. Subagents reserved for discovery. Better quality, less cleanup.

✅ Acceptance-Driven

Define done before starting. BDD format: Given/When/Then for every feature.

Quick Start

Installation

Add to your opencode.json:

{
  "plugins": {
    "npm": ["openpaul"]
  }
}

The Core Loop

PLAN ──▶ APPLY ──▶ UNIFY

Your First Loop

# Initialize OpenPAUL
/openpaul:init

# Create a plan
/openpaul:plan

# Review and approve, then execute
/openpaul:apply

# Close the loop (required!)
/openpaul:unify

Using with OpenCARL (Recommended)

OpenCARL is a dynamic rule injection system that auto-enforces OpenPAUL workflow discipline. Together, they create a powerful AI-assisted development experience.

Install Both

{
  "plugins": {
    "npm": ["openpaul", "@krisgray/opencarl"]
  }
}

Essential Commands

Core Loop

  • /openpaul:init — Initialize project
  • /openpaul:plan — Create plan
  • /openpaul:apply — Execute plan
  • /openpaul:unify — Close loop

Session

  • /openpaul:progress — Status + next action
  • /openpaul:pause — Create handoff
  • /openpaul:resume — Restore context
  • /openpaul:help — Full reference
